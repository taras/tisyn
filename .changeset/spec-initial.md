Add `@tisyn/spec` for authoring, normalizing, indexing, and validating the Tisyn specification corpus. Spec authors can now express specs and test plans through PascalCase constructors, normalize them into hashed canonical artifacts, build a registry with rule/error-code/concept/term indices, and run V1..V9 structural validation plus coverage and readiness queries — all with zero runtime dependencies on other `@tisyn/*` packages.
